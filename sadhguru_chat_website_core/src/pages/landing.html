Sadhguru Flow · SPA · CLI & browser

A quiet space for questions that matter

This site introduces a small AI-assisted project: a Python CLI that talks to models through Ollama, and a vanilla web app where you can try a Sadhguru-style dialogue in your browser with WebLLM (WebGPU). Same intention — different runtimes.

Source code on GitHub

What you’ll find here

The GitHub repository pairs a terminal workflow for power users with a no-backend web demo. You choose: remote Ollama models with your own keys, or fully local inference in Chromium via WebLLM — no account on this site.

  • Documentation SPA — Hash routes (#/landing, #/home, …) swap views without a full reload.
  • Home — Short intro plus the WebLLM chat panel and quick-reference cards.
  • The app / CLI / WebGPU — Deeper notes mirrored from the Python tool and browser stack.

Features

Sadhguru voice (system prompt)

Replies aim for depth, metaphor, and inward focus — not generic assistant tone. The same core prompt drives CLI and web.

CLI + Ollama

Ask once or stay in a REPL-style loop. Point OLLAMA_HOST at your server and pass --model to match a tag your Ollama knows.

Optional CLI modes

--opc “yogic code” phrasing; --sans Sanskrit–Tamil blend with English gloss (CLI only).

WebLLM in the tab

Pick a bundled MLC model, load weights once, then chat offline in spirit — data stays on-device after download.

Vite + npm

The UI is packaged with npm scripts; local dev uses headers that satisfy WebLLM’s COOP/COEP needs.

Single Page App

One HTML shell, client-side routing, and lazy-loaded WebLLM so first paint stays light.

Run the Python CLI

  1. Clone the repo on GitHub and open a terminal at the project root.
  2. Create a virtual environment, then pip install -r requirements.txt (includes ollama, python-dotenv, rich).
  3. Set OLLAMA_HOST and OLLAMA_API_KEY in a .env file (see sadhguru_chat.py).
  4. Ensure your Ollama server exposes the model you pass with --model (default in code may be gpt-oss:20b — adjust to what you host).
  5. Run python sadhguru_chat.py --question "…" or --interactive.

Tip: use --opc or --sans only when you want those experimental prompt modes.

Run this website locally

From the repository root, the Makefile installs dependencies and starts the Vite dev server: 

make start-web

Equivalent:

cd sadhguru_chat_website_core
npm install
npm run dev

Then open the printed URL (typically http://localhost:5173/). Use Home for WebLLM chat; first model load can take a while and uses significant disk in the browser cache.
