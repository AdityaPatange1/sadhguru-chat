Browser chat & WebGPU

WebLLM loads a quantized model into your browser. The first load can take time and disk space; later visits reuse the cache.

A recent Chromium browser with WebGPU enabled is strongly recommended. Firefox and Safari support is evolving — if loading fails, use the CLI path instead.

Security headers (Cross-Origin-Opener-Policy and Cross-Origin-Embedder-Policy) are set in the Vite dev server so WebLLM can use shared memory where needed.

This UI lives in the repo under sadhguru_chat_website_core/ — see the GitHub project for source and issues.
